Custom Skills
Reusable prompts that run with a single /command
Why for you: You repeatedly do website updates (content, styling, images) and deploy to staging. A /deploy skill would bundle commit + push + verify, and a /website-update skill could ensure Claude follows the right workflow every time without re-explaining.
mkdir -p .claude/skills/deploy && cat > .claude/skills/deploy/SKILL.md << 'EOF' # Deploy to Staging 1. Run any pending image compression on files > 1MB 2. Git add all changes 3. Create a descriptive commit message 4. Push to staging branch 5. Confirm push succeeded (do NOT ask user to verify) EOF
Hooks
Shell commands that auto-run at specific lifecycle events
Why for you: You had issues with large uncompressed images (40MB files) getting committed. A pre-commit hook could auto-check for oversized images and compress them, preventing the problem that required a dedicated cleanup session.
# Add to .claude/settings.json: { "hooks": { "pre-commit": { "command": "find . -name '*.png' -o -name '*.jpg' | xargs -I{} sh -c 'size=$(stat -f%z \"{}\" 2>/dev/null || stat -c%s \"{}\"); [ \"$size\" -gt 2000000 ] && echo \"WARNING: {} is $(($size/1048576))MB - consider compressing\"'" } } }
